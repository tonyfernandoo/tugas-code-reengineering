Data Class


  • Apa Itu Data Class?

    • Data Class, merujuk pada kelas dalam sebuah program yang kebanyakan hanya berisi properti atau data tanpa adanya logika bisnis atau fungsi yang signifikan. Dalam konteks code smell, keberadaan Data Class menunjukkan bahwa kelas tersebut mungkin tidak memberikan nilai tambah yang cukup signifikan dalam struktur perangkat lunak.

    Kehadiran Data Class dalam kode sumber dapat mengakibatkan kompleksitas yang tidak perlu dan mempersulit pemeliharaan serta pengembangan lebih lanjut dari aplikasi. Selain itu, mereka dapat menjadi tanda bahwa desain perangkat lunak Anda perlu direfaktorisasi untuk mengurangi redundansi dan meningkatkan kohesi antar kelas.

    Dengan memahami Data Class, Anda dapat mengidentifikasi pola-pola tersebut dalam kode sumber Anda dan mengambil langkah-langkah yang diperlukan untuk meningkatkan kebersihan serta efisiensi dari struktur perangkat lunak yang Anda kembangkan.


  • Tanda-tanda Data Class

    • Bagaimana kita tahu jika kode kita terdapat adanya Data Class? Berikut adalah beberapa tanda yang dapat membantu Anda mengidentifikasi keberadaannya:

    1. Hanya Berisi Properti: Jika sebuah kelas hanya terdiri dari beberapa properti tanpa adanya logika atau perilaku tambahan yang terkait, ini bisa menjadi indikasi bahwa kelas tersebut adalah Data Class.

    2. Tidak Ada Logika Bisnis: Jika kelas hanya berfungsi sebagai wadah untuk data tanpa melakukan operasi yang berhubungan dengan bisnis atau domain aplikasi, ini bisa menjadi tanda bahwa kelas tersebut adalah Data Class.

    3. Tidak Ada Ketergantungan: Jika kelas tersebut memiliki ketergantungan yang minim atau tidak signifikan terhadap kelas lain dalam proyek, pertimbangkan apakah kelas tersebut benar-benar diperlukan atau bisa digabungkan dengan kelas lain yang lebih relevan. Ketergantungan yang minim bisa menjadi indikasi bahwa kelas tersebut hanya berfungsi sebagai Data Class.

    Dengan memperhatikan tanda-tanda ini, Anda dapat mengidentifikasi keberadaan Data Class dalam kode sumber Anda dan mengambil langkah-langkah yang diperlukan untuk meningkatkan kebersihan dan efektivitas dari struktur perangkat lunak yang Anda kembangkan.


  • Strategi untuk Mengatasi Data Class

    • Mengatasi Data Class dalam kode sumber merupakan langkah penting untuk meningkatkan kebersihan, kohesi, dan efektivitas dari proyek perangkat lunak Anda. Berikut adalah beberapa strategi yang dapat Anda terapkan untuk mengatasi Data Class:

    1. Refaktorisasi Kode: Langkah pertama dalam mengatasi Data Class adalah dengan melakukan refaktorisasi kode. Tinjau ulang kelas-kelas yang diidentifikasi sebagai Data Class dan pertimbangkan untuk menggabungkannya dengan kelas lain yang relevan atau memindahkan logika bisnis yang terkait ke dalam kelas yang lebih sesuai. Dengan cara ini, Anda dapat mengurangi kompleksitas kode dan meningkatkan kohesi antar kelas.

    2. Penerapan Metode-Metode Operasi: Jika sebuah kelas hanya berisi properti tanpa adanya operasi atau logika tambahan, pertimbangkan untuk menambahkan metode-metode operasi yang relevan ke dalam kelas tersebut. Misalnya, Anda dapat menambahkan metode untuk memproses atau memanipulasi data yang disimpan dalam kelas, sehingga kelas tersebut tidak hanya berfungsi sebagai wadah untuk data tetapi juga memiliki fungsionalitas yang berguna.

    3. Pemisahan Fungsionalitas: Jika sebuah kelas memiliki tanggung jawab yang berlebihan atau mencoba melakukan terlalu banyak hal sekaligus, pertimbangkan untuk memisahkan fungsionalitasnya ke dalam kelas-kelas yang lebih spesifik. Dengan memisahkan fungsionalitas yang berbeda ke dalam kelas-kelas yang lebih kecil dan terfokus, Anda dapat meningkatkan modularitas dan keterbacaan kode Anda.

    4. Penggunaan Desain Berorientasi Objek yang Lebih Baik: Evaluasi desain berorientasi objek dari proyek Anda dan pastikan bahwa kelas-kelas Anda memiliki tanggung jawab yang jelas dan terdefinisi dengan baik. Gunakan prinsip-prinsip desain berorientasi objek seperti SOLID untuk memastikan bahwa setiap kelas memiliki satu tujuan yang jelas dan memenuhi prinsip-prinsip desain yang baik.

    Dengan menerapkan strategi ini, Anda dapat mengatasi Data Class dalam kode sumber Anda dan meningkatkan kebersihan serta efektivitas dari proyek perangkat lunak yang Anda kembangkan. Teruslah memperhatikan pola-pola yang tidak efisien dalam kode Anda dan berusaha untuk memperbaiki mereka demi mencapai struktur kode yang lebih baik dan lebih mudah dipelihara.


  • Contoh Kasus

    • Ketika produk baru awal-awal di-code, sangat wajar bila banyak class yang belum memiliki behavior (baru isi data saja). Namun, bila produk sudah makin berkembang, namun masih ada suatu class yang hanya berisi setter/getter, sudah sinyal kuat untuk dilakukan refactor.

    Pada contoh kasus FullName.java, class hanya berisi setter getter untuk field firstName dan lastName.

    Before

    FullName.java :

    
package fowler.dispensables.data_class.before;

public class FullName {
	private String firstName;
	private String lastName;
	
	public FullName(String firstName, String lastName) {
		super();
		this.firstName = firstName;
		this.lastName = lastName;
	}
	
	public String getFirstName() {
		return firstName;
	}
	public void setFirstName(String firstName) {
		this.firstName = firstName;
	}
	public String getLastName() {
		return lastName;
	}
	public void setLastName(String lastName) {
		this.lastName = lastName;
	}
}

    User.java

    
package fowler.dispensables.data_class.before;

public class User {
	private FullName fullName;
	
	private String address;
	private String phone;
	
	//...
	//another User behaviors...
	//...
	
	public void login() {
		//...
	}
	
	public void logout() {
		//...
	}
}

    After

    User.java :

    
package fowler.dispensables.data_class.after;

public class User {
	private String firstName;
	private String lastName;
	
	private String address;
	private String phone;
	
	//...
	//another User behaviors...
	//...
	
	public void login() {
		//...
	}
	
	public void logout() {
		//...
	}
}


    Semua field dari FullName dipindahkan ke User.java. Kemudian class FullName dihapus.

    Atau tambahkan method-method untuk melengkapi Data Class jika memerlukan fungsionalitas. Anda juga bisa biarkan smell ini terjadi bila Data Class tersebut difungsikan untuk keperluan Domain-Driven Programming terutama pada level Repository atau Model.
