Parallel Inheritance Hierarchies


  • Apa Itu Parallel Inheritance Hierarchies?

    • Parallel Inheritance Hierarchies adalah sebuah pola desain yang terjadi ketika kita memiliki dua hirarki kelas yang berjalan sejajar (parallel) satu sama lain. Salah satu hirarki kelas adalah untuk kelas dasar (base class), sementara yang lainnya untuk kelas turunan (subclasses). Hal ini mengakibatkan setiap kali ada perubahan di satu hirarki, kita sering kali harus melakukan perubahan yang sesuai di hirarki lainnya. Dengan kata lain, struktur kode menjadi terkait erat antara dua hierarki, menyebabkan ketergantungan yang kuat dan kompleksitas yang tidak perlu. Sebagai contoh, kita mungkin memiliki hirarki kelas untuk entitas seperti kendaraan dan yang lainnya untuk merek kendaraan. Jika kita ingin menambahkan merek baru, kita harus mengubah kedua hirarki tersebut, menciptakan Parallel Inheritance Hierarchies.

    Dalam Parallel Inheritance Hierarchies, perubahan di satu hirarki cenderung memicu serangkaian perubahan yang kompleks di hirarki lainnya. Hal ini dapat mengarah pada kode yang sulit dipelihara, rentan terhadap kesalahan, dan sulit untuk dimengerti. Oleh karena itu, penting bagi para pengembang perangkat lunak untuk mengenali pola ini dan mencari cara untuk menghindarinya. Dengan memahami esensi dari Parallel Inheritance Hierarchies, kita dapat menemukan strategi untuk memperbaiki desain kode yang lebih baik, lebih modular, dan lebih mudah dipelihara.


  • Tanda-tanda Parallel Inheritance Hierarchies

    • Beberapa tanda yang menunjukkan adanya Parallel Inheritance Hierarchies dalam kode adalah pola ketergantungan yang saling terkait antara dua hirarki kelas. Anda mungkin melihat bahwa setiap kali terjadi perubahan di satu hirarki, perubahan yang sesuai juga harus dilakukan di hirarki lainnya. Selain itu, kode mungkin menampilkan tingkat kompleksitas yang tinggi karena ketergantungan yang rumit antara dua hierarki tersebut. Hal ini bisa menjadi indikator adanya Parallel Inheritance Hierarchies yang memerlukan perhatian lebih dalam dalam desain ulang kode.


  • Strategi untuk Mengatasi Parallel Inheritance Hierarchies

    • Untuk mengatasi Parallel Inheritance Hierarchies, teknik refactoring seperti "Move Method" dan "Move Field" dapat digunakan untuk memisahkan ketergantungan antara dua hirarki kelas yang berjalan sejajar. Dengan "Move Method", kita dapat memindahkan metode yang terkait secara erat dengan satu hirarki ke hirarki lainnya, mengurangi ketergantungan dan meningkatkan modularitas kode. Selain itu, dengan "Move Field", atribut yang berkaitan dengan satu hirarki dapat dipindahkan ke hirarki lainnya, memungkinkan kedua hirarki untuk berdiri sendiri tanpa terlalu terkait. Dengan menerapkan kedua teknik ini, kita dapat membersihkan dan merapikan struktur kode, meningkatkan fleksibilitas, dan mengurangi kompleksitas yang tidak perlu.


  • Contoh Kasus

    • Kita memiliki dua hirarki kelas yang saling terkait, yaitu "Vehicle" (Kendaraan) dan "Brand" (Merek). Setiap kendaraan memiliki mereknya sendiri, dan setiap merek memiliki kendaraan yang terkait dengannya. Namun, logika untuk menampilkan merek kendaraan yang terkait masih tersebar di kedua hirarki kelas tersebut.

    Before
    
class Vehicle {
  private Brand brand;
                  
  public String getBrandName() {
      return brand.getName();
  }
}
                  
class Brand {
  private String name;
                  
  public String getName() {
      return name;
  }
}

    Dalam kode di atas, logika untuk mendapatkan nama merek kendaraan ("getBrandName") terdapat di kelas "Vehicle", sedangkan logika untuk mendapatkan nama merek itu sendiri ("getName") terdapat di kelas "Brand". Ketergantungan saling terkait antara dua hirarki kelas ini menyebabkan kode mengalami Parallel Inheritance Hierarchies.


    Kita dapat menggunakan teknik "Move Method" untuk memindahkan metode ("getBrandName") dari kelas "Vehicle" ke kelas "Brand", sehingga logika untuk mendapatkan nama merek kendaraan terpusat dalam satu hirarki kelas.

    After
    
class Vehicle {
  private Brand brand;
                  
  // Menghapus metode getBrandName
}
                  
                  class Brand {
  private String name;
                  
  public String getBrandName() {
      return name;
  }
}


    Setelah menggunakan teknik "Move Method", metode ("getBrandName") sekarang menjadi bagian dari kelas "Brand". Ini menghilangkan Parallel Inheritance Hierarchies karena logika untuk mendapatkan nama merek kendaraan terpusat dalam satu hirarki kelas. Dengan demikian, kode menjadi lebih bersih, terstruktur, dan mudah dipelihara.
