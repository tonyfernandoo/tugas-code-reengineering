Dead Code

Ketika persyaratan untuk perangkat lunak telah berubah atau koreksi telah dilakukan, tidak ada yang punya waktu untuk membersihkan kode lama.
Kode tersebut juga dapat ditemukan dalam kondisi kompleks, ketika salah satu cabang menjadi tidak dapat dijangkau.

Treatment

1. Hapus kode yang tidak digunakan dan file yang tidak diperlukan
2. Jika subclass atau superclass digunakan
- Inline Class
- Collapse Hierarchy
3. Untuk menghapus parameter yang tidak diperlukan
- Remove Parameter


  • Contoh Kasus

    • Awalnya requirement dari client adalah: diskon diberikan 10% bila harga kurang dari 10.000, jika lebih, diskon 20%. Terbentuklah code berikut:

    Before

    
public double calculate(double price, boolean isDiscount){
  double discountPrice = 0;

  if(isDiscount){
    if(price < 10000){
      discountPrice = price * 0.1;
    } else {
      discountPrice = price * 0.2;
    }
  }

  return price - discountPrice;
}

    Tiba-tiba, requirement dari client berubah. Client meminta diskon diketok rata 15%.

    Programmer tidak menghapus if yang pertama. Ia langsung menambahkan di bagian bawah sebelum return. Secara requirement, ini tidak masalah. Diskon akan selalu 15%.


    
public double calculate(double price, boolean isDiscount){
  double discountPrice = 0;

  if(isDiscount){
    if(price < 10000){
      discountPrice = price * 0.1;
    } else {
      discountPrice = price * 0.2;
    }
  }

  if(isDiscount)
    discountPrice = price * 0.15;
  return price - discountPrice;
}

    After

    PriceCalculator.java :

    
package fowler.dispensables.dead_code.after;

public class PriceCalculator {
	public double calculate(double price, boolean isDiscount){
		double discountPrice = 0;
	
		if(isDiscount) discountPrice = price * 0.15;
		
		return price - discountPrice;
	}
}


    Variabel discountPrice akan selalu ditimpa di line of code selanjutnya. Sehingga bagian yang tidak penting aman untuk dibuang.
