Dead Code


  • Apa Itu Dead Code?

    • Dead Code, adalah istilah yang digunakan dalam pengembangan perangkat lunak untuk merujuk pada bagian-bagian dari kode sumber yang tidak lagi digunakan atau dieksekusi dalam aplikasi. Dalam konteks code smell, Dead Code merupakan salah satu indikasi bahwa terdapat fragmen kode yang tidak memberikan kontribusi fungsional apa pun kepada aplikasi.

    Penyebab Dead Code bisa bermacam-macam, mulai dari refaktorisasi kode yang tidak sempurna hingga perubahan kebutuhan fungsional aplikasi.

    Mengidentifikasi Dead Code adalah bagian penting dari pemeliharaan kode yang baik. Dengan membersihkan Dead Code dari proyek Anda, Anda dapat meningkatkan kebersihan, keterbacaan, dan kinerja keseluruhan dari aplikasi Anda.


  • Tanda-tanda Dead Code

    • Bagaimana kita tahu jika kode kita terdapat adanya Dead Code? Berikut adalah beberapa tanda yang dapat membantu Anda mengidentifikasi keberadaannya:

    1. Fungsi atau Metode Tidak Dipanggil: Perhatikan fungsi atau metode yang didefinisikan namun tidak pernah dipanggil dari bagian lain dalam kode. Fungsi-fungsi ini mungkin merupakan bagian dari implementasi lama yang tidak lagi diperlukan atau telah digantikan oleh fungsionalitas baru.

    2. Variabel Tidak Digunakan: Tinjau variabel yang dideklarasikan tetapi tidak pernah digunakan dalam aliran eksekusi program. Variabel yang tidak digunakan ini sering kali merupakan sisa-sisa dari pengembangan sebelumnya atau kesalahan dalam logika program.

    3. Kode yang Tidak Diakses: Perhatikan modul, kelas, atau bagian-bagian kode lain yang tidak pernah diakses dalam aplikasi. Kode-kode ini mungkin merupakan sisa-sisa dari iterasi pengembangan sebelumnya yang tidak lagi relevan dengan kebutuhan aplikasi saat ini.

    Dengan memperhatikan tanda-tanda ini, Anda dapat mengidentifikasi Dead Code dalam kode sumber Anda dan mengambil langkah-langkah yang diperlukan untuk membersihkannya. Dengan membersihkan Dead Code secara teratur, Anda dapat menjaga kebersihan dan kualitas dari kode Anda, serta meningkatkan efisiensi dan keterbacaan dari proyek perangkat lunak Anda.


  • Strategi untuk Mengatasi Dead Code

    • Mengatasi Dead Code dalam kode sumber merupakan langkah penting untuk meningkatkan kebersihan, keterbacaan, dan performa dari aplikasi Anda. Berikut adalah beberapa strategi yang dapat Anda terapkan untuk mengatasi Dead Code:

    1. Analisis Kode yang Teliti: Lakukan analisis kode yang teliti untuk mengidentifikasi bagian-bagian kode yang tidak aktif atau tidak digunakan. Tinjau setiap fungsi, metode, variabel, dan modul untuk memastikan bahwa mereka benar-benar digunakan dalam aliran eksekusi program.

    2. Pembersihan Kode yang Rutin: Tetapkan jadwal rutin untuk membersihkan Dead Code dari proyek Anda. Buat daftar potensial kode yang tidak digunakan dan identifikasi mana yang dapat dihapus atau dinonaktifkan.

    3. Pemantauan Perubahan Kode: Perhatikan setiap perubahan yang terjadi dalam proyek Anda dan pastikan bahwa Dead Code tidak muncul kembali. Manfaatkan alat bantu seperti sistem kontrol versi untuk melacak perubahan kode dan memeriksa adanya penambahan Dead Code.

    4. Pengujian dan Validasi:Setelah menghapus atau menonaktifkan Dead Code, lakukan pengujian menyeluruh untuk memastikan bahwa perubahan tersebut tidak menyebabkan masalah baru dalam aplikasi Anda. Pastikan untuk menguji berbagai skenario penggunaan untuk memverifikasi bahwa aplikasi masih berfungsi dengan baik.

    Dengan menerapkan strategi ini secara konsisten, Anda dapat mengatasi Dead Code dalam kode sumber Anda dan meningkatkan kebersihan serta efektivitas dari proyek perangkat lunak yang Anda kembangkan. Teruslah memantau dan memperbarui kode Anda secara teratur untuk menghindari akumulasi Dead Code dan memastikan bahwa aplikasi Anda tetap efisien dan mudah dipelihara.


  • Contoh Kasus

    • Awalnya requirement dari client adalah: diskon diberikan 10% bila harga kurang dari 10.000, jika lebih, diskon 20%. Terbentuklah code berikut:

    Before

    
public double calculate(double price, boolean isDiscount){
  double discountPrice = 0;

  if(isDiscount){
    if(price < 10000){
      discountPrice = price * 0.1;
    } else {
      discountPrice = price * 0.2;
    }
  }

  return price - discountPrice;
}

    Tiba-tiba, requirement dari client berubah. Client meminta diskon diketok rata 15%.

    Programmer tidak menghapus if yang pertama. Ia langsung menambahkan di bagian bawah sebelum return. Secara requirement, ini tidak masalah. Diskon akan selalu 15%.


    
public double calculate(double price, boolean isDiscount){
  double discountPrice = 0;

  if(isDiscount){
    if(price < 10000){
      discountPrice = price * 0.1;
    } else {
      discountPrice = price * 0.2;
    }
  }

  if(isDiscount)
    discountPrice = price * 0.15;
  return price - discountPrice;
}

    After

    PriceCalculator.java :

    
package fowler.dispensables.dead_code.after;

public class PriceCalculator {
	public double calculate(double price, boolean isDiscount){
		double discountPrice = 0;
	
		if(isDiscount) discountPrice = price * 0.15;
		
		return price - discountPrice;
	}
}


    Variabel discountPrice akan selalu ditimpa di line of code selanjutnya. Sehingga bagian yang tidak penting aman untuk dibuang.
