Duplicate Code


  • Apa Itu Duplicate Code?

    • Duplicate Code, dalam pengembangan perangkat lunak, merujuk pada keberadaan fragmen kode yang sama atau serupa yang muncul di beberapa bagian dari program. Ini terjadi ketika pengembang menyalin dan menempelkan kode yang serupa di beberapa lokasi, tanpa memperhatikan untuk mengkonsolidasikan atau menggabungkannya ke dalam satu entitas yang dapat digunakan kembali. Kondisi ini mengakibatkan pengulangan yang tidak perlu dalam kode, yang pada gilirannya dapat menyebabkan penurunan kualitas, pemeliharaan yang sulit, dan risiko kesalahan yang meningkat. Dalam praktiknya, Duplicate Code sering kali menjadi sumber ketidakefisienan dan kerentanan yang dapat dihindari dengan menerapkan praktik-praktik refactoring yang sesuai.


  • Tanda-tanda Duplicate Code

    • Ada beberapa tanda-tanda yang dapat membantu mengidentifikasi keberadaan Duplicate Code dalam kode Anda. Pertama, perhatikan adanya fragmen kode yang serupa atau identik yang muncul di beberapa bagian dari program. Jika Anda menemukan diri Anda menyalin dan menempelkan kode yang sama di berbagai tempat, ini mungkin merupakan indikasi adanya Duplicate Code. Selain itu, perhatikan apakah ada fungsi-fungsi yang memiliki implementasi yang sama atau serupa, tetapi digunakan untuk tujuan yang berbeda. Ketika Anda menemukan pola-pola seperti ini, kemungkinan besar Anda berurusan dengan Duplicate Code. Jika kode Anda sulit dimengerti karena banyaknya pengulangan atau kesamaan yang tidak perlu, ini juga bisa menjadi tanda adanya Duplicate Code yang perlu diatasi.


  • Strategi untuk Mengatasi Duplicate Codedengan Refactoring

    1. Pull Up Constructor Body: Menggunakan teknik ini, Anda dapat menarik bagian dari logika konstruktor yang sama dari beberapa kelas turunan ke dalam kelas induk mereka. Ini membantu menghindari pengulangan kode dan mengurangi kemungkinan kesalahan.

    2. Form Template Method: Ketika Dengan Form Template Method, Anda dapat membuat metode template yang umum untuk mengurangi kesamaan dalam implementasi metode di beberapa kelas. Ini memungkinkan Anda untuk mempromosikan pola perilaku yang serupa ke dalam metode induk.

    3. Substitute Algorithm: Teknik ini mengganti algoritma yang rumit atau tidak efisien dengan versi yang lebih sederhana atau efisien. Ini membantu mengurangi keberadaan kode yang sama di beberapa bagian dari program.

    4. Consolidate Conditional Expression: Dengan menggabungkan beberapa ekspresi kondisional menjadi satu, Anda dapat mengurangi pengulangan kondisi yang sama di beberapa bagian kode. Ini membuat kode lebih bersih dan mudah dimengerti.

    5. Consolidate Duplicate Conditional Fragment: Teknik ini membantu menggabungkan fragmen kondisional yang serupa atau identik ke dalam satu blok kode. Dengan demikian, Anda dapat menghindari pengulangan kode yang sama di beberapa tempat dan meningkatkan keterbacaan kode secara keseluruhan.

    Dengan menerapkan berbagai teknik refactoring ini, Anda dapat mengatasi Duplicate Code dalam kode Anda dan meningkatkan kualitas serta keterbacaannya. Setiap teknik memiliki kegunaan dan penerapan yang berbeda, sesuaikan dengan kebutuhan dan situasi kode Anda.


  • Contoh Kasus

    • Berikut adalah contoh kasus dalam bentuk kode Java yang mengalami pengulangan kondisional dan kita akan menyelesaikannya dengan menggunakan teknik "Consolidate Conditional Expression":

    Kode berikut mengalami pengulangan kondisional yang sama di beberapa metode, mengakibatkan fragmen kode yang serupa di beberapa tempat.

    Before
    
public class Customer {
  private boolean isNewCustomer;
  private boolean isRegularCustomer;
                  
  public double calculateDiscount(double totalPrice) {
      if (isNewCustomer) {
          if (totalPrice > 100) {
              return totalPrice * 0.1;
          } else {
              return totalPrice * 0.05;
          }
      } else {
          if (isRegularCustomer && totalPrice > 100) {
              return totalPrice * 0.15;
          } else {
              return totalPrice * 0.1;
          }
      }
  }
                  
  // Other methods with similar conditional logic...
}

    Kode di atas memiliki fragmen kondisional yang serupa di dua tempat, yaitu pada metode "calculateDiscount" dan metode lain yang tidak ditunjukkan. Pengulangan ini mengakibatkan pengulangan kode yang tidak efisien dan sulit dipelihara.


    Kita dapat menggunakan teknik "Consolidate Conditional Expression" untuk mengatasi pengulangan kondisional tersebut dengan menggabungkan kondisi-kondisi yang serupa ke dalam satu blok kode.

    After
    
public class Customer {
  private boolean isNewCustomer;
  private boolean isRegularCustomer;
                  
  public double calculateDiscount(double totalPrice) {
      double discount = 0.0;
      if (isNewCustomer && totalPrice > 100) {
          discount = totalPrice * 0.1;
      } else if (isNewCustomer) {
          discount = totalPrice * 0.05;
      } else if (isRegularCustomer && totalPrice > 100) {
          discount = totalPrice * 0.15;
      } else {
          discount = totalPrice * 0.1;
      }
      return discount;
  }
                  
  // Other methods...
}

    Setelah menerapkan teknik "Consolidate Conditional Expression", kita menggabungkan kondisi-kondisi yang serupa ke dalam satu blok kode. Ini membuat kode menjadi lebih bersih, lebih mudah dimengerti, dan menghindari pengulangan kondisional yang tidak perlu. Dengan cara ini, kita meningkatkan kualitas dan pemeliharaan kode secara keseluruhan.
