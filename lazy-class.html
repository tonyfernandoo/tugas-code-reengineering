Lazy Class


  • Apa Itu Lazy Class?

    • Lazy Class, adalah sebuah konsep dalam pengembangan perangkat lunak yang mengacu pada kelas atau objek yang memberikan kontribusi yang minim atau tidak signifikan terhadap fungsionalitas keseluruhan aplikasi. Dalam konteks code smell, Lazy Class merupakan salah satu pola yang menandakan adanya kode yang tidak efisien atau kurang terorganisir dengan baik.

    Keberadaan Lazy Class dalam sebuah proyek perangkat lunak dapat mengakibatkan kompleksitas yang tidak perlu, membingungkan pembaca kode, dan menghambat pemeliharaan serta pengembangan lebih lanjut dari aplikasi.

    Mengidentifikasi Lazy Class dalam kode sumber, pengembang dapat meningkatkan kualitas dan kebersihan dari struktur perangkat lunak mereka, memfasilitasi pemeliharaan yang lebih mudah, dan mempercepat proses pengembangan secara keseluruhan.


  • Tanda-tanda Lazy Class

    • Bagaimana kita tahu jika kode kita terdapat adanya Lazy Class? Berikut adalah beberapa tanda yang dapat membantu Anda mengidentifikasi keberadaannya:

    1. Kelas dengan Sedikit Metode: Ketika sebuah kelas hanya memiliki satu atau dua metode yang tidak melakukan banyak hal atau bahkan tidak digunakan sama sekali, ini mungkin menjadi indikasi adanya Lazy Class.

    2. Kelas yang Hanya Menjadi Tempat Penyimpanan Data: Jika sebuah kelas hanya berfungsi sebagai wadah untuk menyimpan data tanpa adanya perilaku atau fungsionalitas tambahan yang signifikan, ini bisa menjadi tanda bahwa kelas tersebut merupakan Lazy Class.

    3. Kelas dengan Ketergantungan Minim: Jika sebuah kelas memiliki sedikit atau bahkan tidak ada ketergantungan pada kelas lain di dalam proyek, ini bisa menjadi tanda bahwa kelas tersebut tidak berperan penting dalam sistem secara keseluruhan.

    Dengan memahami tanda-tanda ini, Anda dapat lebih mudah mengidentifikasi keberadaan Lazy Class dalam kode Anda dan mengambil langkah-langkah untuk mengatasinya.


  • Strategi untuk Mengatasi Lazy Class

    • Mengatasi Lazy Class merupakan langkah penting dalam meningkatkan kebersihan dan efisiensi dari kode sumber Anda. Berikut adalah beberapa strategi yang dapat Anda terapkan untuk mengatasi Lazy Class dalam proyek perangkat lunak Anda:

    1. Refaktorisasi Kode: Salah satu pendekatan utama untuk mengatasi Lazy Class adalah dengan melakukan refaktorisasi kode. Tinjau kembali kelas-kelas yang diidentifikasi sebagai Lazy Class dan pertimbangkan untuk menggabungkannya dengan kelas lain yang terkait atau memindahkan fungsionalitasnya ke kelas yang lebih relevan. Dengan melakukan refaktorisasi, Anda dapat menyederhanakan struktur kode dan menghilangkan redundansi yang tidak perlu.

    2. Analisis Ketergantungan: Lakukan analisis mendalam terhadap ketergantungan antar kelas dalam proyek Anda. Identifikasi kelas-kelas yang memiliki ketergantungan minimal atau tidak signifikan terhadap kelas lainnya, dan evaluasi apakah kelas-kelas tersebut benar-benar diperlukan. Jika tidak, pertimbangkan untuk menghapus atau mengonsolidasikan kelas-kelas tersebut untuk mengurangi kompleksitas kode.

    3. Uji Kode: Sebelum melakukan perubahan besar pada kode, pastikan untuk menguji perubahan tersebut secara menyeluruh. Buat unit tes yang mencakup skenario pengujian untuk memvalidasi perubahan yang Anda buat, dan pastikan tidak ada perubahan yang merusak fungsionalitas aplikasi. Pengujian yang cermat akan membantu memastikan bahwa setiap perubahan yang Anda buat tidak menyebabkan masalah baru dalam proyek.

    4. Gunakan Alat Bantu:Manfaatkan alat bantu seperti linter kode atau perangkat lunak analisis statik untuk membantu mengidentifikasi Lazy Class dalam kode sumber Anda. Alat-alat ini dapat memberikan wawasan tambahan tentang struktur kode dan memberikan rekomendasi untuk perbaikan. Namun, selalu penting untuk melakukan peninjauan manual terhadap hasil analisis alat bantu untuk memastikan keakuratan dan relevansinya.

    Dengan menerapkan strategi ini, Anda dapat mengatasi Lazy Class dalam kode sumber Anda dan meningkatkan kebersihan serta efisiensi dari proyek perangkat lunak yang Anda kembangkan. Teruslah memperhatikan kode Anda dan berusaha untuk memperbaiki pola-pola yang tidak efisien atau tidak perlu demi mencapai struktur kode yang lebih baik dan lebih mudah dipelihara.


  • Contoh Kasus

    • Lazy class bisa saja terjadi karena refactoring. Awalnya class ini memiliki fungsi yang banyak. Namun, satu per satu dipindahkan ke class lain karena pertimbangan tertentu.

    Pada contoh di PriceValidator.java. Class hanya memiliki satu buah fungsi untuk validasi harga.

    Before

    Price.java :

    
package fowler.dispensables.lazy_class.before;

public class Price {
	private int value;

	public Price(int value) throws Exception {
		if(!PriceValidator.validate(value)) {
			throw new Exception("price not valid");
		}
		this.value = value;
	}

	public int getValue() {
		return value;
	}
}

    PriceValidator.java

    
package fowler.dispensables.lazy_class.before;

public class PriceValidator {
	public static boolean validate(int value){
		return value >= 0;
	}
}

    After

    Price.java :

    
package fowler.dispensables.lazy_class.after;

public class Price {
	private int value;
	
	public Price(int value) throws Exception {
		if(!isPriceValid(value)) {
			throw new Exception("price not valid");
		}
		
		this.value = value;
	}

	private boolean isPriceValid(int value){
		return value >= 0;
	}
	
	public int getValue() {
		return value;
	}
}


    Fungsi validate dipindahkan ke class Price.java kemudian diberi nama yang lebih sesuai untuk class Price: isPriceValid. Setelah dipindahkan, class PriceValidator bisa dihapus.
